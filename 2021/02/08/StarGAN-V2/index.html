StarGAN V2

学习论文：StarGAN v2: Diverse Image Synthesis for Multiple Domains

论文目的：It propose a new version of StarGAN that can generate diverse images and scalability over mutiple domains.

关键词： Diversity Multi-Domain

introduction

这篇文章结构有些诡异，主要内容是style的定义，一个域中的图片有着同一个分类，但是他们各自不同，这就是他们的风格。

Here, domain implies a set of images that can be grouped as a visually distinctive category, and each image has a unique appearance, which we call style。

但是其他人定义是 domain里大家共享的特征，但是这样就限制了多样性。（我更倾向他）。

还记得StarGAN的内容就是输入了一个label，但是这样也就限制了它的多样性，所以本文就采取了style code编码同时本文还提出了一个新的数据集，AFHQ。

这是本文的在多样性和多域的表现。

这篇文章好像失去了多数据集了。

正文

网络结构

首先是网络结构，这个就复杂了，分成4个部分.

首先是Generator，输入图片和style code ，输出图片。

Mapping network，输入一个latent code z,生成多个域的相关编码style code。

Style encoder ，这个就是输入一个input image，把他编码成style code相关。

Discriminator，输出多个域上的真假，最后只选择一个。

Loss

这里他竟然神奇的有了4个网络，我看着觉得666。

  1. adversarial loss，这里就没啥特别的地方，这里的$s$是有F生成的 s=F(z)，z取自高斯分布。这里F和G是一个目标

  2. Style reconstruction Loss：这个loss是为了尽可能充分利用这个style code，同时E也可以解码训练图片，一一举两得。

  3. Style diversification Loss:这个loss我是真没想到，这个我觉得很不好，为了鼓励不同，因为原本的z1-z2的loss不稳定，所以改用这个。

  4. Preserving source characteristics，保存身份信息的

  5. 总的loss，这里注意一个负号，因为要尽可能多样化麻，所以为了转max为min加了个负号。

表现

首先是BaseLine ，MUNIT [13], DRIT [22], and MSGAN。

其次是评价指标 Frechét inception distance (FID) [11] and learned perceptual image patch similarity (LPIPS)。

直接看效果图：好的甚至让我觉得在换脸，特别是同性别间。

这里存在一个疑问latent-guided synthesis and reference-guided synthesis.这个是啥啊，原文也没有介绍。

论文中介绍这个latent-guided synthesis能做到头发的风格迁移，amazing。

reference-guided synthesis也能做到很好的效果相比于baseline。

总结成功的理由：

  1. 首先得益于style code，使得G可以专心学习利用这些信息。
  2. 其次是和StyleGAN一样，它只想学习转换规则，这相比于baseline就多了很多弹性。
  3. 最后，由于可以多域学习，所以就可以扩大数据集的使用。

代码解读

G

这人代码写的真是赛博朋克，我看哭了。直接上四大部分。

首先是G，这里设计了一个ResBlk和AdainResBlk,这里就没啥看可以解释的，我也不单独拎出来了，但是这个写法着实骚，类似U-Net，但是没有U-Net的跨接，这里还有一个mask的操作，在论文里面没有看到。

MappingNetwor

这里就是把高斯噪声变成一个stylecode，估计是一个MLP的过程。也真的还是有多个MLP。一个域就一个MLP，然后根据y去获得对应的MLP。

这里要介绍torch.cat的一个兄弟，一个 torch.stack，这个函数和它唯一的区别就是会增加一个维度，把序列的要求拼接的维度升维。

其次是torch.LongTensor，这个和tensor有啥区别了，其实就类似于指明具体类型的tensor，注意Tensor其实就是FloatTensor的别名。

StyleEncoder

这个写的虽然魔幻，但是实际就是上面的conv版，没啥好多说的。

D

这里不同于原来的D，但只是输出多域的信息，还是没有啥区别。
